**Workflow** (per [P0-9] cookie maintenance design):
1. Quit Chrome completely (Chrome single-user-data-dir lock)
2. Launch Chrome with `--remote-debugging-port=9222`
3. In Chrome: log in to your proxy → navigate to www.cnki.net (any page)
4. Run this script → exports `.cnki.net` cookies to `~/.paper-agent/cookies/cnki.json`
5. Verify with `pa cnki status` + `pa search "东数西算" --engine cnki`

**Implementation details**:
- PowerShell 5.1+ compatible (uses `Invoke-WebRequest` for Chrome CDP check)
- Auto-checks Python + playwright availability
- Auto-checks Chrome debug port reachability (4-step recovery if not)
- Uses `playwright.chromium.connect_over_cdp()` (Playwright bundled Chromium works;
  tested with Edge 150.0.4078.65 via CDP — Chrome should also work since both
  are Chromium-based and CDP is standard)
- Filters cookies by `cnki.net` domain substring
- Saves in Playwright `context.add_cookies()` JSON format (matches what
  `pa_cli/cnki_channel.py.load_cookies()` reads)
- In-process temp file (`%TEMP%\_pa_export_cnki_cookies.py`) generated + cleaned up
  on exit, to avoid PowerShell interpolation issues with Python quotes / `{}` / f-strings
- Graceful exit codes: 0=success, 1=python/playwright missing, 2=Chrome unreachable,
  3=no CNKI cookies found, 4=playwright CDP error
