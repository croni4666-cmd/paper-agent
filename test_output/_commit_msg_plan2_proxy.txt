## Discovery: proxy flows need multi-domain cookie capture

Previous script version only captured `cnki.net` cookies via substring filter
`"cnki.net"`. This works for direct CNKI users but FAILS for proxy users:
- xueshu789.com (proxy) session cookies are on a DIFFERENT domain (e.g.
  PHPSESSID on www.xueshu789.com)
- cnki.net (target) session cookies (ECP_SESSIONID) are on cnki.net
- BOTH sets are needed: Mavis visits xueshu789.com first, gets redirected to
  CNKI, then cnki.net cookies are set

## Fix: multi-substring filter + smart hint on zero-match

Updated `C:\Users\DengN\.mavis\bin\Export-CNKICookies.ps1` (10123 -> 12250 bytes):

1. **DomainFilter now supports `|` separator**:
   - Default: `"cnki"` (matches cnki.net only, for direct CNKI users)
   - For proxy users: pass `-DomainFilter "cnki|xueshu"` to capture both
     proxy and target cookies

2. **Smart hint on zero-match**:
   - When no cookies found, list ALL domains with cookies in Chrome
   - Detect proxy-like keywords (xueshu, ezproxy, proxy, vpn, lib, edu)
   - Suggest the right `-DomainFilter` invocation

3. **Pass script name via env var** to Python subprocess (avoids PS
   interpolation issues with `$MyInvocation.MyCommand.Name`)
