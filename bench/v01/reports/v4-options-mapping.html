LTR (Learning to Rank)

P0 · 1-2h · 5-10% lift
真实实现
  • 算法:LambdaMART / LightGBM ranker(gradient-boosted trees + pairwise rank loss)
  • 训练目标:objective=lambdarank,优化 NDCG
  • 8 维特征:BM25 / biencoder / combined / PRF / citation_count / year / is_recent / has_abstract
  • 数据:25 queries × 30 candidates × 3-level labels = 3,725 labeled pairs
  • 5-fold CV,per-query group(candidates of same query 必须在同一 fold)
  • 库:lightgbm(pure local,$0)
  • 参考:AkariAsai/OpenScholar 用了同样的 LightGBM 风格 rerank
替代实现 (v3.9.0)
  • combined = 0.5 × BM25 + 0.5 × biencoder
  • 固定线性加权,没有学习
  • 权重 0.5/0.5 是拍脑袋定的(没有数据驱动)
  • 不能识别"BM25 高 + biencoder 低 vs BM25 低 + biencoder 高 哪个更相关"
  • 6 条件之一,跟 bm25 / biencoder / prf / random / original 并列
  • 来自 bench/v01/_v4_rerank.py
差距 / 估时
  • 不能 capture feature 之间的非线性交互
  • 权重手动调,无数据驱动
  • 不能 learn 不同 query type 应用不同 feature weight
1-2h 修复路径 → [P0-6] LambdaMART, 加 200 LOC pa_cli/ltr.py + 5-fold CV 报告
2

Cross-encoder reranker

P0 · 2-3h · 5-15% lift
真实实现
  • 模型:BAAI/bge-reranker-base(~278MB 单 .bin 文件,本地下载一次)
  • 架构:query + candidate 拼接作为单一输入,transformer 内部 L×L 注意力跨 token 交互
  • 推理:batch (query, candidate) pair → relevance score ∈ [0, 1]
  • 流程:bi-encoder top-30 → cross-encoder rerank top-30 → final ranking
  • 库:sentence-transformers.CrossEncoder(HuggingFace)
  • 参考:BGE-reranker 官方 repo, MIT, ~3k stars, IR 文献广泛引用
替代实现 (v3.9.0)
  • Bi-encoder:SentenceTransformer('all-MiniLM-L6-v2') 单独 embed query 和 candidate
  • cosine similarity 作为 score
  • query 和 candidate 没有任何 token 级别 attention
  • 例:query "AI in higher ed" 和 candidate title "Generative AI in higher education: A systematic review" 的 token overlap 没用上
  • 来自 pa_cli/search.py 的 _embed_biencoder()
差距 / 估时
  • 无 query-candidate 跨 token 注意力
  • bi-encoder 更快但精度低 5-15%(标准 IR 经验值)
  • 不能 capture token-level semantic match(同义词 / 改写)
2-3h 修复路径 → [P0-7] BGE-reranker, 新增 pa_cli/cross_encoder.py(一次性下载 278MB 本地)
3

MoE routing (for IR)

P1 · 0.5-1d · 5-10% lift
真实实现
  • 5 个 LGBMClassifier(per-engine,Crossref / S2 / arxiv / OpenAlex / CORE 各一)
  • 训练 label:per query,哪个 engine 贡献了最多 label=2 候选
  • 特征:TF-IDF on query text(max 5000)+ query metadata(length / has-acronym / year constraint)
  • 输出:per-(query, engine) weight ∈ [0, 1],sum = 1
  • 应用:搜索时 per-engine weighted aggregation,而非 round-robin
  • 库:sklearn + lightgbm,纯本地
  • 参考:GitHub 无现成 IR-MoE 库,模式参考 OpenScholar + Mixture-of-Retrievers 论文
替代实现 (v3.9.0)
  • "unweighted MoE":5-engine round-robin pool,固定顺序
  • min_per_source interleaving,无 learned weights
  • 每个 query 都走同样顺序:arXiv → OpenAlex → S2 → Crossref → CORE
  • 无 query-type 感知,技术 CS/ML query 和引用图谱 query 都按同一顺序
  • 来自 pa_cli/search.py 的 _interleave_round_robin()
差距 / 估时
  • 无 query-type 感知:技术 query 应该先 arxiv,引用图谱应该先 Crossref,OA 应该先 CORE
  • round-robin 浪费:technical query 拉了 OpenAlex 但前 5 命中都是 0
  • 所有 query 都消耗同样 5-engine 查询预算
0.5-1d 修复路径 → [P1-11] MoE router, 新增 pa_cli/moe_router.py(routing labels 已有,从 v3.9.0 benchmark 提取)

PaSa-lite (北大的 pa)

P2 · 1-2 周 · partial · 未知 lift
真实实现
  • Crawler agent:7B LLM (Qwen2.5-7B),4 actions — search / read / expand / stop
  • Selector agent:7B LLM,decision token + reasoning chain
  • 训练:SFT(13,000 演示轨迹)+ PPO session-level,16 GPU 数周
  • 外部依赖:Google Search API(serper.dev, paid $)+ arxiv/ar5iv
  • 数据集:AutoScholarQuery(35k 合成 queries),RealScholarQuery benchmark
  • 效果:优于 Google Scholar + GPT-4 paraphrased
  • 来源:ByteDance Seed + 北大鄂维南, arXiv 2501.10120
替代实现 (v3.9.0, rule-based ~50%)
  • Multi-strategy query expansion:pa search --concepts (OpenAlex concept) + --prf (pseudo-relevance feedback) + --expand (synonym/related)
  • 1-hop citation walk:pa citations <DOI> --direction forward(已 shipped,[P1-1])
  • Iterative refinement:2-3 轮 "query → top-30 → re-query using top titles → dedup → rerank"(未实现)
  • 无 LLM reasoning,无 adaptive stop decision
差距 / 估时
  • 无 LLM 驱动的 query 改写(PaSa 的 "expand" action)
  • 无 relevance reasoning(PaSa Selector 读 full paper 决定 relevance)
  • 无 adaptive stop(固定 2-3 轮,LLM 决定 convergence)
  • 不能 content-aware rerank(PaSa 读 full PDF,paper-agent 只有 abstract)
  • PaSa 全版 Global Rule ❌(hosted LLM + paid API)
1-2 周 修复路径 → [P2-6] PaSa-lite rule-based,rule-based 50% 子集可实现;PaSa 全版永久 Global Rule ❌
